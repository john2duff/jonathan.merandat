Introduction
Terminologie
  • Attaque : action de malveillance consistant à détourner les fonctions de sécurité d'un système informatique.
  • Menace : violation potentielle de la sécurité.
  • Risque : combinaison de la probabilité de l'occurence et de la gravité.
  • Vulnérabilité : faute, par malveillance ou maladresse, dans les spécifications, la conception, etc...

Menaces

  • Vol d'information

    ⤷ Données bancaires

  • Création, modification ou destruction non autorisé d'information

    ⤷ Modifié un prix lors d'une commande sur un site e-commerce.
    ⤷ Modifié une note sur la plateforme de formation.

  • Déni de service

    ⤷ Rendre indisponible le système.

  • Ralentissement du service

    ⤷ Perte des performances du système (downtime).

  • Usurpation d'identité

    ⤷ Attaque par spoofing.

  • Répudiation

    ⤷ La non-répudiation est l'impossibilité de nier la participation à un traitement de l'information.
    ⤷ Ex : effacer la certification qu'un message a été émis et reçu.

  • Défiguration

    ⤷ Défiguration d'un site web pour des raisons politiques, criminelles, etc...

Autres types de menaces

  • Logiciels malveillants

    ⤷ Virus
    Portion de code inoffensive ou destructive capable de se reproduire et de se propager.
     Différents types de virus : virus boot, virus dissimulé dans les exécutables.
    Différents types de propagation : par des supports amovibles, pièce jointe mail, par le réseau internet??
    ⤷ Vers
    Proche des virus mais capable de se propager sur d'autres ordinateurs à travers le réseau.
    ⤷ Chevaux de Troie
    Un cheval de Troie est un virus qui vise à donner accès aux ressources de la machine à l'insu de son utilisateur ou envoyer des données.
     Plusieurs types de chevaux de Troie : back door, droppers, proxy, espion.
    ⤷ Logiciels espions (Spyware)
    Malware qui envoie de l'information depuis la machine infectée.
     A la différence des vers, ce type de programme ne se propage pas lui-même mais est installé lors d'une installation logicielle.
    Les logiciels espions (spyware) servent aussi à envoyer des publicités ciblées.
    ⤷ Publiciels
    Logiciels qui présentent des publicités non sollicitées
     L'effet est pénible pour l'utilisateur.
    Ces logiciels adWare accompagnent souvent des logiciels gratuits téléchargé sur le web.
    ⤷ Pirate de navigateur (Browser Hijaker)
    Concerne les modifications du navigateur sans la permission de l'utilisateur pour afficher des publicités.
     Egalement utilisé pour génèrer de fausses alertes de sécurité pour inciter à acheter un anti-virus.

  • Hameçonnage (Phishing)

    ⤷ Vise à faire croire à la victime qu'elle s'adresse à un site de confiance afin de récupérer des renseignements personnels (mot de passe, numéro de carte de crédit, date de naissance, etc...)

  • Botnets (Réseaux zombies)

    ⤷ Ensemble de robots informatiques qui se déploient sur de grandes quantités de machines pour les utiliser comme relais de spam, d'attaques ou autres actions malveillantes.
    ⤷ La mise en place du botnet utilise les malware pour se déployer et l'attaquant contrôle le réseau via P2P ou HTTP.

  • Exploitation de vulnérabilités

    ⤷ Consiste des dans morceaux de code exploitant des failles connues dans les navigateurs.

  • Ingénierie sociale (Spam)

    ⤷ Exploite la naïveté des utilisateurs de systèmes informatiques pour récupérer des informations.

Risques

Criticité = Gravité * Probabilité

Il existe des risques au niveau réseau, systèmes, et applications.

Propriété de sécurité à préserver :

  • Confidentialité Seuls les acteurs autorisés peuvent voir l'information.
  • Intégrité Seuls les acteurs autorisés peuvent créer, modifier et supprimer l'information.
  • Disponibilité Les acteurs autorisés peuvent avoir accès à l'information quand ils le veulent.
  • Auditabilité Il est possible de tracer les évènements sur le système.

Attaquants
Les Hackers agissent pour différentes raisons :
  • Défi intellectuel
  • Liberté d'expression et d'information
  • Cause politique ou religieuse
  • Cyber-criminalité / banditisme
  • Espionnage d'entreprise ou d'Etat
  • Cyberguerre

Attaques
Injection SQL

Une injection SQL vise à modifier une requête pour effectuer une action non autorisée.

Mise en oeuvre

$sql = "SELECT * FROM users WHERE username='$username' AND password='$password' ";
⤷ $sql = "SELECT * FROM users WHERE username=''a' OR '1'='1' --' AND password=''";

On peut également utiliser des commandes d'insertion, de suppression, etc... :

$sql = "SELECT * FROM users WHERE username='$username' AND password='$password' ";
⤷ $sql = "SELECT * FROM users WHERE username='';INSERT INTO users (username,password) VALUES ('bb', 'bf');--'' AND password=''";
⤷ $sql = "SELECT * FROM users WHERE username='';DROP TABLE users;--'' AND password=''";
Protection contre les injections SQL
  1. Filtrer les caractères spéciaux contenus (ex : fonction php -> mysql_real_escape_string)
  2. Utilisation d'expression régulières pour filtrer les entrées avant l'appel SQL.
  3. Utilisation de requête préparée (paramétrée).
Cross site scripting(XSS)

Il existe deux types d'attaque XSS :
  • Stored XSS : le code malicieux est stocké sur le serveur.
  • Reflected XSS : le code malicieux est réfléchi sur un poste client sans stockage sur le serveur.
Mise en oeuvre
Modification d'un lien
http://www.example.com/search.php?query=products
est remplacé par
http://www.example.com/search.php?query=< script >alert(document.cookie)< /script >
Protection contre le XSS
Ne pas remonter les données non fiables au navigateur
  1. Sur le serveur applicatif, filtrer les données non fiablesbasées sur le contenu HTML (corps, attribut, Javascript, CSS ou URL)
  2. Sur le poste client au niveau navigateur, valider les entrées en vérifiant la longueur des champs ou le type de données.
  3. Utilisation d'un encodage pour encapsuler les caractères spéciaux dans les parties non fiables.

Analyse de risque de sécurité
Processus de gestion des risques ISO 27005:2011
Etablissement du contexte

On cherche à établir le contexte externe (environnement culturel, social, politique, les relations avec les parties prenantes, etc...) et interne (l'organisation, les rôles, les capacités en termes de ressources, les normes, etc...)

Appréciation des risques

Souvent réalisé en deux itérations. Une appréciation de haut niveau afin d'identifier les risques majeurs potentiels puis l'analyse plus détaillés de ces risque.

Traitement des risques

Sert à identifier les mesures pour réduire, maintenir, éviter ou transférer les risques.

Communication relative aux risques

Il est important de bien communiquer les risques et les enjeux aux parties prenantes afin qu'elle puisse prendre des décisions en connaissance de causes. Il est important que leurs perceptions des risques ainsi que leurs bénéfices soient bien documentée.

Surveillance et revue des risques
Les risques ne sont pas statiques, une surveillance constante est donc nécessaire concernant les points suivants :
  • les nouveaux actifs ayant été inclus dans la gestion des risques
  • les modifications des valeurs des actifs
  • les nouvelles menaces susceptibles d'être actives.
  • l'augmentation du risque (probabilité ou gravité)
RSSI

Responsable de la sécurité des Systèmes d'Information est chargé de la définition et de la mise en oeuvre de la politique de sécurité de l'entreprise. Suivant la taille de l'organisation, le RSSI peut encadrer une équipe d'ingénieurs et techniciens.

Vulnérabilités par injection XSS et SQLi
Cross-Site Scripting ou XSS

Le Cross-Site Scripting ou XSS est une attaque par injection qui exploite les interactions entre une application Web et ses utilisateurs.

Un attaquant peut par exemple :

  • Voler une session en récupérant le cookie et prendre possession du compte
  • Diffuser des vers (comme Sammy Worm dans MySpace)
  • Accéder à l'historique du navigateur
  • Contrôler le navigateur à distance
  • Scanner et exploiter des applications intranet
  • Scanner les fichiers stockés sur le poste client
  • Installer un Trojan
  • Rediriger l'utilisateur vers un site externe
  • Défigurer un site

Exemple d'injection :
<script>XX = [ICI]</script> <img src="[ICI]" /> <div style=[ICI]:bla> <style>div{'aa:a[ICI]aa'}</style> <body onload:"[ICI]"> </body> <div onmouseover:'XX(1,'[ICI]')'></div>

Et la liste n'est pas du tout exhaustive!
